## Highlights

CodingBar v1.1.1 ships **in-app auto-update**. After this version, you no
longer have to download a `.dmg` from the website and right-click → Open
for every release.

### How it works

- Open Settings (the gear in the top-right of the panel) and flip
  **"自动检查更新" / "Auto-check for updates"** on. It's **off by default**.
- CodingBar then quietly checks `appcast.xml` from GitHub Releases once a
  day. When a new version is out, the Settings row turns into a
  **"立刻更新 →" / "Update now →"** button — one click, Sparkle downloads
  the new build, replaces `/Applications/CodingBar.app` in place, and
  restarts.
- macOS Gatekeeper will **not** prompt again on auto-installed updates —
  the prompt only ever appears on the very first manual install.

Powered by [Sparkle 2.x](https://sparkle-project.org), the de-facto Mac
update framework used by VSCode, CleanShot, Bartender and most other
non–App Store apps.

### Why opt-in (and why you should opt in)

The auto-check is **off by default** so the original privacy promise
("no automatic network calls outside the quota path") still holds for
anyone who never enables it. But unless you have a reason not to, you
probably want it on — otherwise you'll miss security/bug fixes the same
way you have been.

## Security — EdDSA signatures, not Apple's chain

Every update is cryptographically signed with an **EdDSA private key**
held only in this repo's GitHub Actions secret. The matching public key
is hard-coded in `Info.plist`. Sparkle refuses to install anything whose
signature doesn't verify against that public key — so even if someone
hijacked the GitHub Release or swapped the zip, the update would fail
closed.

This is also why CodingBar still works **without** an Apple Developer ID
($99/yr) — Sparkle's trust anchor is our key, not Apple's notarization.

## Privacy boundary — updated

The v1.1.0 notes said "the only automatic network reads are quota". That
promise now has an opt-in extension:

- Quota: Claude / Codex usage endpoints (user's own OAuth token, 5-min TTL).
- **(opt-in)** Sparkle: small `appcast.xml` GET once a day + an update
  zip download when you confirm. EdDSA-verified, no auth, no telemetry,
  no local data uploaded.

Off by default. Toggleable from Settings. Source: `Sources/CodingBar/UpdateManager.swift`.

## Install

**Existing users (v1.1.0 and earlier)**: this one upgrade still has to be
done manually — download `CodingBar-v1.1.1.dmg` below and replace your
`/Applications/CodingBar.app`. Future versions can install themselves
once you flip the toggle on.

**New users**: download the `.dmg`, drag CodingBar to `/Applications`,
right-click → Open the first time to bypass Gatekeeper (ad-hoc signed,
no Developer ID).
